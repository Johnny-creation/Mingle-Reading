# 名家 Agent 思维方式评测：方法论设计

> 目标：为"我们的名家 agent 真正捕捉了名家的**内在思维方式**，而不只是**语言风格**的模仿"这一命题，提供可复现、可证伪、不依赖人工专家标注的量化证据。

本目录 (`backend/eval/persona_thinking/`) 是一套独立评测，不改动既有的 `run_eval.py`（那套是 schema 校验 + 关键词匹配的 demo 级评测，与"思维保真度"无关）。

---

## 1. 研究问题与构念界定

要证明的命题里有两个必须先拆开的构念：

| 构念 | 定义 | 如何度量 |
|---|---|---|
| **语言风格 (style)** | 词汇、句法、修辞、标点的表层特征（文白夹杂、破折号、华丽辞藻、中英夹杂、"倘若…然而…"句式…） | `stylometry.py`，确定性的表层特征统计 |
| **思维方式 (thought)** | 面对**新输入**时的潜在决策过程：关注什么、用什么价值透镜、做哪些推理跳转、如何框定问题 | `rubrics/*.json` 的思维维度，由裁判对**去风格后**的文本打分 |

**关键论点**：思维只在"给定新材料时得出的判断与框定"里暴露，与"怎么说"无关。因此唯一有判别力的检验是——喂入语料里没有的新情境，看 agent 的**判断/框定**是否仍是该名家会有的，且**独立于风格**。

---

## 2. 承重设计：风格–内容解耦 (style–thought dissociation)

整套证明的核心是一个对照实验。对每道探题，在三种条件下生成回答：

| 条件 | system prompt | 含义 |
|---|---|---|
| `full` | 完整 `SKILL.md`（思维框架 + 风格） | 产品实际部署的名家 agent |
| `style_only` | **只有语言风格**，删除全部思维动作/价值立场/方法论 | 关键对照组：纯风格模仿者 |
| `neutral` | 普通助手，无人设 | 地板参照 |

随后对三者的回答做**风格剥离**（`style_strip.py`：改写成平实中性语体，保留全部论点/推理/对照的实质，只改"怎么说"），再用思维 rubric 给**剥离后的文本**打分。

> **判读**：如果在风格被剥离之后，`full` 在思维 rubric 上仍显著高于 `style_only`，那么这部分差距**不可能由风格解释**，只能归因于思维。这就是"捕捉到思维而非风格"的直接证据。
>
> 反过来，配套的 `style_only` 基线是一个**能让命题失败的对照**——如果纯风格模仿就能拿到同样的思维分，命题在该名家上就**不成立**。评测因此是可证伪的。

剥离针对的是**语体/句法**（文白、破折号、句式、辞藻），而**概念性的认知动作**（包括用隐喻去蔽这类"用比喻揭示结构"的推理）被有意保留——因为那是思维，不是风格。

---

## 3. 用语料锚定真值，而非裁判的刻板印象

没有领域专家标注时，最大的风险是裁判"凭它脑子里对鲁迅的印象"打分。缓解办法：把名家**真实原文**作为评分锚点。

`tools/extract_anchors.py` 从 `《鲁迅全集》` / `张爱玲大全集` 的 EPUB 中抽取真实正文片段（鲁迅：《拿来主义》《论"费厄泼赖"应该缓行》《我之节烈观》《狂人日记》等；张爱玲：《自己的文章》《金锁记》《倾城之恋》《天才梦》《红玫瑰与白玫瑰》等），存入 `corpus_anchors/*.json`。裁判打分时，这些片段作为"该作家**实际如何思考**的证据"一并给出。这样把"裁判对名家的想象"换成了"裁判对照名家真实文本"。

> 因为探题是**语料外的当代新现象**（算法、内卷、网络围观…），锚点提供的不是话题匹配，而是**思维动作的范例**——裁判判断的是 agent 是否把名家可证实使用过的认知动作，迁移到了新话题上。这正是"思维迁移"的恰当检验。

---

## 4. 指标与判读

`run.py` 汇总输出（`results/result_*.md` / `.json`）：

1. **核心指标——思维分差 `full − style_only`**（风格剥离后，1-5）。> 0 且明显 = 思维被捕捉。
2. **强制选择胜率**：把 `full` 与对照组的（去风格）回答随机排成 A/B 给裁判盲选"哪个思维更像该名家"，统计 `full` 胜率。该指标对打分量纲不敏感，更稳健。`full` vs `style_only` 是关键，`full` vs `neutral` 是下限检查。
3. **风格被控制的证据**（`stylometry.py`）：展示 `full` 与 `style_only` 的风格特征**接近**、且都明显高于 `neutral`。这证明风格已被复制并在两条件间持平，从而思维分差**不能**用风格差异搪塞。
4. **逐维度分**：定位 agent 在哪些具体思维动作上强/弱。

---

## 5. 两层效度

- **内部保真度**：`full` 是否忠实执行了 `SKILL.md` 写定的思维模型？→ rubric 直接度量。
- **外部效度**：`SKILL.md` 这套思维模型本身是否**真的等于**那位名家？→ 由"语料锚定"承担：rubric 高分样貌来自 SKILL，而裁判同时对照真实原文，二者一致才给高分。

只证内部保真，只说明"造出了所设计的东西"；外部效度（对照真实语料）才支撑"它真的是鲁迅"。

---

## 6. 信度与威胁，及缓解

| 威胁 | 说明 | 缓解 |
|---|---|---|
| **裁判自评偏差** | 当前被测 agent 与裁判都是 DeepSeek（同族），裁判可能偏好同族输出 | ①语料锚定（对照真实文本而非凭感觉）②盲评（裁判不知条件）③强制选择随机 A/B 消除位置偏差。**根治办法**：换跨族裁判（Claude/GPT）——在 `common.py` 改 `JUDGE_ENV` 即可，架构已支持多裁判 |
| **单裁判信度** | 暂只用 DeepSeek 一个裁判 | `--judge-runs N` 多次评分，报告自一致性（总分标准差）。多模型交叉一致性是后续替代人工信度的最强证据 |
| **风格剥离改动内容** | 剥离时可能误删论点 | 低温改写 + 强约束"只改怎么说不改说什么"；已人工抽检确认论点保留（见 §8） |
| **探题泄露思维** | 探题或 style_only prompt 若含思维线索会污染对照 | style_only prompt 已剔除一切思维/价值措辞（仅留语体）。张爱玲早期版本因含"画面/苍凉"被发现并修正——见下 |

---

## 7. 可证伪性是特性，不是 bug（以张爱玲为例）

首轮小规模结果：**鲁迅** `full−style_only ≈ +1.8`、强制选择 `full` 全胜——分离干净；**张爱玲** gap ≈ 0、强制选择约五五开。

这不是失败，而是评测**有判别力**的证明：

- 鲁迅的思维是一种独特的**论证方法**（二难推理、归谬、历史循环、去蔽），通用文学腔调产生不出来——所以 SKILL 的思维框架带来真实增益。
- 张爱玲的思维偏**感知/审美**，且基座模型在"分析都市男女场景"时本就会自然带出物质细节、心理、权力视角；因此"仅风格"已能拿到相近思维分，SKILL 的**边际思维贡献在当前设置下未被证明**。

正确做法是**如实报告**，并讨论成因（基座先验 / rubric 区分度 / 探题更贴合她的天然取向），而不是反复调参直到张爱玲"过关"——后者是 p-hacking，恰是答辩应警惕的。一个不能失败的评测，证明不了任何东西。

---

## 8. 复现方式

```bash
# 1) 一次性：从 EPUB 抽取语料锚点
python backend/eval/persona_thinking/tools/extract_anchors.py

# 2) 全量评测（两名家、8 题/人、2 轮评分）
python -m backend.eval.persona_thinking.run --personas lu-xun zhang-ailing --judge-runs 2

# 小规模冒烟：
python -m backend.eval.persona_thinking.run --personas lu-xun --max-probes 2 --judge-runs 1
```

- 配置：`.env`（工作区根目录）提供 DeepSeek 凭据；agent 走各名家端点，裁判/剥离走 `MUSE_NEUTRAL_*`。
- 所有模型调用磁盘缓存于 `_cache/`，重跑不重复烧 token；改 prompt 会因 key 变化自动重算相关项。
- 结果写入 `results/result_<时间戳>.{json,md}`。

## 9. 已知局限与后续

1. **裁判同族**——最优先：接入 Claude / GPT 作跨族裁判，报告多裁判一致性。
2. **样本量**——每名家 8 题，宜扩到 ~30 题并做显著性检验（如对 `full` vs `style_only` 配对的符号检验 / bootstrap 置信区间）。
3. **rubric 区分度**——张爱玲维度偏"通用文学解读"，可增设更专属、更难的探题与维度。
4. **可加 `ablated`（仅思维无风格）第四条件**，与 `style_only` 形成完整 2×2，进一步分离风格与思维的独立贡献。
5. **风格剥离的保真**——可加一道"内容一致性"自动校验（裁判判断剥离前后论点是否一致）。
